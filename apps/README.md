# USC App Repositories

Concrete systems built under USC discipline should live outside this substrate repo.

Use this naming pattern:

```text
narada.usc.<app-name>
```

Examples:

```text
narada.usc.helpdesk
narada.usc.ops-console
narada.usc.sonar-operator
```

An app repo may contain product code, private decisions, domain-specific task graphs, operational sessions, and deployment configuration. Reusable construction semantics should be extracted back into `narada.usc`.
